In Module: 1 of the Accessible Name &amp; Description Inspector (ANDI) Tool video series, get an overview of the Accessible Name & Description Inspector (ANDI) tool, a web accessibility inspection tool, which performs single page testing to automatically detect accessibility issues.

  <div id="a1" class="usa-accordion__content">
    <p>Voice Over (VO): The Interagency Trusted Tester Program is hosted by the Department of Homeland Security's Office of Accessible Systems and Technology.</p>
    <p>VO: Welcome to the basic overview of the Accessible Name and Description Inspector tool, also known as ANDI.</p>
    <p>VO: ANDI is a web accessibility inspection tool which performs single-page testing to automatically detect accessibility issues.</p>
    <p>VO: Once successfully installed, ANDI will be seen in the Favorites or Bookmarks bar. For demonstration purposes, we are using Firefox.</p>
    <p>VO: The ANDI icon will look slightly different depending on which browser you are using.</p>
    <p>VO: ANDI is compatible with Mozilla Firefox, Google Chrome, Internet Explorer, Microsoft Edge, and Safari.</p>
    <p>VO: To launch ANDI, simply select the ANDI icon and it will open at the top of your browser.</p>
    <p>VO: It is recommended to use the ANDI version mentioned in the Trusted Tester Tools installation course. To check the version you are using, select the ANDI icon.</p>
    <p>VO: ANDI is comprised of multiple modules which include:</p>
    <ul>
      <li>Focusable Elements</li>
      <li>Graphics and Images</li>
      <li>Links and Buttons</li>
      <li>Tables</li>
      <li>Structures</li>
      <li>Color Contrast</li>
      <li>Hidden Content</li>
      <li>iFrames</li>
    </ul>
    <p>VO: These can be accessed using the drop-down menu. The modules displayed will vary depending upon the page content.</p>
    <p>VO: When selecting a new module, ANDI rescans the page and displays results.</p>
    <p>VO: We will now describe the purpose of each module.</p>
    <p>VO: The Focusable Elements module analyzes interactive elements that can be accessed using the keyboard.</p>
    <p>VO: The Graphics and Images module analyzes all graphics and images on the page.</p>
    <p>VO: The Links and Buttons module analyzes all links and buttons on the page.</p>
    <p>VO: The Tables module allows testers to analyze layout and data tables. If there are no tables on the page, this module will not appear.</p>
    <p>VO: The Structures module identifies programmatically determinable headings, lists, page language, reading order, live regions, and page title.</p>
    <p>VO: Select the More Details menu to continue to identify page title, page language, and language attributes.</p>
    <p>VO: The Color Contrast module analyzes the contrast between text and the background. ANDI may recommend you perform a manual test of the contrast using a Color Contrast Analyzer tool.</p>
    <p>VO: The Hidden Content module exposes hidden content.</p>
    <p>VO: The iFrame module tests for the presence of iframes on the page. iFrames must be opened in a new window to test their content. If there are no iframes on the page, this module will not appear.</p>
    <p>VO: Controls and Settings buttons are part of the ANDI toolbar and include Refresh, Event Settings, Hotkeys, Help, and Close.</p>
    <p>VO: The Refresh button rescans the page in the current module. Testers must use it any time there is any change in page content.</p>
    <p>VO: The Advanced Settings button opens the menu with element highlights, linearized page, and mini mode options.</p>
    <p>VO: The Hotkeys List button displays a list of hotkeys that are unique to ANDI.</p>
    <p>VO: The Help button opens Help topics related to the current module.</p>
    <p>VO: The Close button closes ANDI and any markup it provided to the web page.</p>
    <p>VO: Caution: selecting the browser's Refresh button or pressing `F5` refreshes the page and closes ANDI.</p>
    <p>VO: This concludes the ANDI overview. Users are encouraged to view the other videos for more information on how to utilize ANDI to test web content.</p>
