In Module 3 of the Accessible Name &amp; Description Inspector (ANDI) Tool video series, understand how to use ANDI’s _Links/Buttons_ module to inspect [interactive][1] links and buttons to evaluate for accessibility so users will know where the link will take them if selected.

  <div id="a1" class="usa-accordion__content">
    <p>Voice Over (VO): Welcome to the ANDI Links and Buttons module. The purpose of this module is to inspect interactive links and buttons for accessibility.</p>
    <p>VO: To start, open a web page and launch ANDI. Next, choose the Select module drop-down menu and navigate to the Links and Buttons module.</p>
    <p>VO: Once selected, the Links buttons menu display the number of links and buttons at the top of the page. To evaluate links, select the Links menu.</p>
    <p>VO: Once selected, all the links are highlighted with a dotted blue line. Next, select the previous element or next element button to navigate the links on the page.</p>
    <p>VO: Or, select the plus sign from the View Links list to view and navigate the complete list of links. Once the link receives focus, it is highlighted with a solid pink outline.</p>
    <p>VO: The ANDI output will display the programmatic markup. Evaluate the ANDI output to validate it accurately conveys the purpose of the link.</p>
    <p>VO: To evaluate buttons, select the Buttons menu. Once selected, all the buttons are highlighted with a dotted blue line.</p>
    <p>VO: Next, select the previous element or next element button to navigate the buttons on the page. Or, select the plus sign from the View Buttons list to view and navigate the complete list of buttons.</p>
    <p>VO: Once a button receives focus, it is highlighted with a solid pink outline. The ANDI output displays the programmatic markup.</p>
    <p>VO: Evaluate the ANDI output to validate it accurately conveys the purpose of the button. A link or button passes if the ANDI output displays adequate programmatic markup that describes the control's purpose.</p>
    <p>VO: A link or button fails if the ANDI output does not display adequate programmatic markup that describes the control's purpose. The test result does not apply when there are no links or buttons found on the page.</p>
    <p>VO: This concludes the ANDI links and buttons module. Users are encouraged to view the other videos for more information on how to utilize ANDI.</p>
