<h2 style="line-height:1.2;">Want five quick fixes to make your documents more accessible?</h2>

You can avoid some of the most common accessibility issues in electronic documents if you:

<ol class="usa-process-list">
  <li class="usa-process-list__item">
    <h4 class="usa-process-list__heading">Use sufficient text alternatives for non text content.</h4>
    <p class="margin-top-05">Using sufficient text alternatives for non-text content helps ensure accessibility by providing meaningful descriptions that can be interpreted by screen readers, enabling users with visual impairments to understand and engage with the content.</p>
    <p>Watch our video "<a href="{{site.baseurl}}/training/alt-text/what-is-alternative-text/">What is Alternative Text?</a>" for a quick introduction to creating meaningful alt text, and visit <a href="{{site.baseurl}}/create/alternative-text/">Authoring Meaningful Alternative Text</a> for a more comprehensive guide to describing images and other non-text content.</p>
  </li>
  <li class="usa-process-list__item">
    <h4 class="usa-process-list__heading">Proceed to the second step</h4>
    <p>Use built-in styles. Using built-in styles such as headings, lists and tables in electronic documents improves accessibility and organization by creating a hierarchical structure that is easily navigable, enhancing overall readability and user experience.</p>
    <p>For more information on styles, see these modules within the Documents training videos:</p>
    <ul>
      <li><a href="{{site.baseurl}}/create/documents/aed-cop-docx02/" target="_blank" class="usa-link--external">Module 2: Use Styles to Create Headings</a></li>
      <li><a href="{{site.baseurl}}/create/documents/aed-cop-docx03/" target="_blank" class="usa-link--external">Module 3: Use Built-in Features to Create Lists</a></li>
      <li><a href="{{site.baseurl}}/create/documents/aed-cop-docx04/" target="_blank" class="usa-link--external">Module 4: Use Built-in Features to Organize Content</a></li>
      <li><a href="{{site.baseurl}}/create/documents/aed-cop-docx05/" target="_blank" class="usa-link--external">Module 5: Use Built-in Features to Create Layout and Data Tables</a></li>
      <li><a href="{{site.baseurl}}/create/documents/aed-cop-docx07/" target="_blank" class="usa-link--external">Module 7: Create Unambiguous Names for Links</a></li>
    </ul>
  </li>
  <li class="usa-process-list__item">
    <h4 class="usa-process-list__heading">Create links with clear, descriptive text and avoid saying "Click Here," or "Read More."</h4>
    <p>This also aligns with web content management best practices for Search Engine Optimization (SEO), where SEO and Section 508 accessibility standards both emphasize the use of clear, descriptive link text to improve user experience and ensure content is easily understood and navigable by both search engines and users, including those with disabilities.</p>
    <p>Discover how <a href="{{site.baseurl}}/blog/accessibility-bytes/descriptive-links-and-hypertext/" target="_blank" class="usa-link--external">descriptive hyperlinks</a> support users who navigate by links alone, helping them quickly find relevant, meaningful content without confusion.</p>
  </li>
  <li class="usa-process-list__item">
    <h4 class="usa-process-list__heading">Use sufficient color contrast for text.</h4>
    <p>Sufficient <a href="{{site.baseurl}}/blog/accessibility-bytes/2/">color contrast</a> between text and its background enhances readability and accessibility, ensuring that content is easily perceivable by all users, including those with visual impairments.</p>
  </li>
  <li class="usa-process-list__item">
    <h4 class="usa-process-list__heading">Avoid using color only</h4>
    <p>Ensure <a href="{{site.baseurl}}/blog/accessibility-bytes/3/">color is not the only visual means of conveying information</a>, indicating an action, prompting a response, or distinguishing a visual element. When you use color to convey information, use other methods, such as text or iconography (with sufficient alternative text, of course) to convey the same information.</p>
  </li>
</ol>
